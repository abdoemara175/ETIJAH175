# GitHub Pages Setup Guide

## تفعيل GitHub Pages للمشروع

اتبع الخطوات التالية لنشر الموقع على GitHub Pages:

### الخطوة 1: الذهاب إلى إعدادات المستودع

1. اذهب إلى: https://github.com/abdoemara175/ETIJAH175
2. انقر على تبويب **Settings** (الإعدادات)

### الخطوة 2: تفعيل GitHub Pages

1. من القائمة الجانبية اليسرى، ابحث عن **Pages** تحت قسم "Code and automation"
2. تحت **Source**، اختر:
   - **Branch**: `main`
   - **Folder**: `/ (root)`
3. انقر على **Save**

### الخطوة 3: انتظر النشر

- GitHub سيبدأ في بناء الموقع تلقائياً
- قد يستغرق 1-2 دقيقة
- ستظهر رسالة "Your site is live at: `https://abdoemara175.github.io/ETIJAH175/`"

### الخطوة 4: الوصول إلى الموقع

بعد التفعيل، سيكون الموقع متاحاً على:

```
https://abdoemara175.github.io/ETIJAH175/
```

## ملاحظات مهمة

✓ جميع الملفات موجودة وجاهزة للنشر  
✓ `index.html` هو الملف الرئيسي  
✓ لا توجد متطلبات بناء (Build) - الموقع يعمل مباشرة  
✓ جميع الملفات (HTML, CSS, JS, JSON) مدرجة في المستودع  

---

## استكشاف الأخطاء

### إذا لم يظهر الموقع:

1. تأكد من أن الفرع (Branch) هو `main`
2. تأكد من أن المجلد (Folder) هو `/` (root)
3. انتظر 2-3 دقائق إضافية
4. حاول تحديث الصفحة (Ctrl+F5 أو Cmd+Shift+R)
5. امسح ذاكرة التخزين المؤقتة (Cache)

### إذا ظهرت رسالة خطأ 404:

1. تأكد من أن `index.html` موجود في جذر المستودع
2. تحقق من أن اسم الملف صحيح (حساس لحالة الأحرف)
3. أعد تحميل الصفحة بعد دقائق قليلة

## خطوات إضافية (اختيارية)

### إضافة نطاق مخصص (Custom Domain)

1. في إعدادات Pages، أضف نطاقك تحت **Custom domain**
2. أضف سجلات DNS في مزود النطاق الخاص بك
3. انتظر تحديث DNS (قد يستغرق 24 ساعة)

### تفعيل HTTPS

- GitHub Pages يفعل HTTPS تلقائياً
- تأكد من أن "Enforce HTTPS" مفعل في الإعدادات

## التحديثات المستقبلية

أي تغييرات تقوم بها على الملفات ستنعكس تلقائياً على الموقع بعد دفعها (Push) إلى GitHub:

```bash
git add -A
git commit -m "تحديث الموقع"
git push origin main
```
